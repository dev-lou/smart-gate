🎯 YOLO11n Uniform Training Guide

Train an AI to detect your school's uniforms — right on your PC

📌 The Big Picture — Where Things Go

Before we start, understand where your files end up:

📸
Your photos: Stay on YOUR PC inside this repo → services/kiosk/training/dataset/
They are temporarily uploaded to Roboflow just for labeling, then you download them back to your PC. After that you can delete them from Roboflow.
🏷️
Label files (.txt): Also on YOUR PC, same folder as the photos. Each photo gets a .txt file that says where the uniform is in the photo.
🤖
Trained model (.onnx): Goes to the cloud (Vercel or Supabase Storage) — this is the 6MB file the tablet downloads. It does NOT contain your photos, just math patterns.
☁️
What goes to Supabase: NOT your photos. Only the uniform names (e.g. "BSIT Uniform" → class_id: 0). These are text records, not images.
💡 Remember: Your photos are ONLY on your PC + temporarily on Roboflow. They NEVER go to Supabase. Only the final 6MB model file goes online.

📸 Step 1: Take Photos

100-200 photos per uniform type

Take photos of students wearing each uniform using your phone. You need both the photo AND a label file for training.

Uniform Type Photos Needed Put in folder
BSIT Uniform 100-200 dataset/images/train/ (80%) + dataset/images/val/ (20%)
CHM Uniform 100-200 Same split
COAGRI Uniform 100-200 Same split
Education Uniform 100-200 Same split

📷 Photo Tips:

  • Different angles (front, side)
  • Different lighting (indoor, outdoor)
  • Different backgrounds (hallway, classroom)
  • Full upper body visible
  • Not blurry
⬇️

🏷️ Step 2: Label Photos on Roboflow

Free, browser-based

Roboflow is where you upload photos, draw boxes around the uniform, and download back with labels. Photos go to Roboflow → you label → you download back with labels included.

2.1 Sign up

  1. Go to roboflow.com — sign up (free tier)
  2. Click "Create Project"
  3. Name: smart-gate-uniforms
  4. Type: Object Detection (Bounding Box)

2.2 Upload your photos

  1. Click "Upload"
  2. Drag & drop your photos from your phone/PC
  3. Set split: 80% train / 20% val
  4. Click "Upload & Continue"

2.3 Draw boxes (label)

  1. Click "Start Annotating"
  2. For each photo: press B → draw a box around the uniform top → select the correct class name
  3. Press right arrow → next photo

2.4 Download back to your PC

  1. Go to "Generate" → click "Generate"
  2. Click "Download"
  3. Format: "YOLO v8"
  4. Click download → you get a ZIP file
💡 The ZIP contains BOTH photos AND labels: When you download from Roboflow, you get:
images/train/photo001.jpg ← the original photo
labels/train/photo001.txt ← the label (where the uniform is)
You need BOTH for training. The image shows YOLO what a BSIT uniform looks like. The .txt file tells YOLO WHERE in the photo it is.
⬇️

📂 Step 3: Place Files in Your Repo (LOCAL)

This is on YOUR PC — inside the project

Extract the ZIP from Roboflow. Then copy the folders into your project's dataset folder. Here's exactly where each file goes:

services/kiosk/training/dataset/ ├── images/ │ ├── train/ ← 📸 Put 80% of your photos HERE │ │ ├── bsit_001.jpg │ │ ├── chm_001.jpg │ │ └── ... │ └── val/ ← 📸 Put 20% of your photos HERE │ ├── bsit_050.jpg │ └── ... ├── labels/ │ ├── train/ ← 🏷️ Put .txt label files that MATCH the train photos HERE │ │ ├── bsit_001.txt │ │ ├── chm_001.txt │ │ └── ... │ └── val/ ← 🏷️ Put .txt label files that MATCH the val photos HERE │ ├── bsit_050.txt │ └── ... └── uniform_dataset.yaml ← ⚙️ This file already exists, just verify class names
⚠️ IMPORTANT: Each photo MUST have a matching .txt file with the SAME name. Example:
bsit_001.jpg → needs bsit_001.txt
bsit_002.jpg → needs bsit_002.txt
If a photo has no .txt file, YOLO will ignore it or error.
📥 How to do this from Roboflow ZIP:
1. Extract the ZIP → you'll see train/ and val/ folders
2. Inside those: images/ and labels/ folders
3. Copy these directly into services/kiosk/training/dataset/
4. Overwrite the existing .gitkeep files
5. That's it — you're ready to train
⬇️

💻 Step 4: Install Requirements

Open a terminal on your PC and run:

cd smart-gate/services/kiosk/training
pip install -r requirements.txt

This installs ultralytics + onnxruntime. Wait for it to finish (~2-5 minutes).

⚡ Step 4.5: (Optional) FP16 Quantization for Faster Tablet Performance

Recommended for tablets

After training your model in Step 5, you can quantize it to FP16 (half precision). This makes the ONNX file ~50% smaller (~3MB instead of ~6MB) and runs ~2x faster on tablet GPUs, with minimal accuracy loss (~1-2%).

💡 When to do this: After Step 5 finishes training, run the export script. The export script already includes FP16 by default.

How precision formats compare:

FormatSizeSpeedAccuracy
FP16 (default)~6 MB~2x faster~99%
FP32 (full)~12 MBBaseline100%
INT8 (quantized)~3 MB~3x faster~95-98%
✅ No extra work needed: The export_to_onnx.py script automatically exports with FP16. Run:
python export_to_onnx.py
The --fp16 flag is enabled by default, creating a ~6MB model optimized for tablet inference.
⬇️

🤖 Step 5: Train the Model (runs on YOUR PC)

Training runs locally — photos never leave your PC during this step

Run the training script:

python train_uniform_yolo.py

What happens:

  1. Downloads YOLO11n pre-trained model (once, ~6MB from internet)
  2. Reads your photos + labels from dataset/ folder
  3. Trains for 100 rounds (epochs) — takes ~30-60 min on CPU, 5-15 min on GPU
  4. Saves best model to runs/train/uniform_detector/weights/best.pt
  5. Exports to ONNX format: runs/train/uniform_detector/weights/best.onnx
💡 What's happening inside the training?
YOLO looks at each photo and its label file. It learns: "When I see a blue shirt with a specific collar and logo, that's class 0 (BSIT). When I see a green shirt, that's class 1 (CHM)." After seeing hundreds of examples, it creates a mathematical model that can recognize uniforms it's never seen before — just like how you can recognize a friend's face even if you've never seen them in that exact lighting.

📊 Good results:

MetricGood Score
mAP50> 85%
Precision> 85%
Recall> 85%

☁️ Step 6: Upload Model to the Cloud

Only the .onnx file (6MB) — no photos

After training, you have a file: runs/train/uniform_detector/weights/best.onnx

This is the ONLY file that goes online. Your photos stay on your PC.

Option A: Put in kiosk public folder (for Vercel)

# Copy the ONNX file:
cp runs/train/uniform_detector/weights/best.onnx ../public/models/uniform_yolo11n.onnx

Then update services/kiosk/src/lib/uniform.ts line 37:

const YOLO_MODEL_URL = "/models/uniform_yolo11n.onnx";

Option B: Upload to Supabase Storage

  1. Supabase Dashboard → Storage → create bucket models (public)
  2. Upload best.onnx
  3. Copy the public URL → update uniform.ts
⬇️

🗄️ Step 7: Update Supabase with Uniform Names

The training script prints the class names and their order. For example:

Class 0: BSIT_Uniform
Class 1: CHM_Uniform
Class 2: COAGRI_Uniform
Class 3: Education_Uniform

Update the migration file database/migrations/002_uniform_types.sql with the EXACT same order:

INSERT INTO uniform_types (name, description, class_id) VALUES
    ('BSIT Uniform', 'BSIT department uniform', 0),
    ('CHM Uniform', 'CHM department uniform', 1),
    ('COAGRI Uniform', 'COAGRI department uniform', 2),
    ('Education Uniform', 'Education department uniform', 3);
⚠️ CRITICAL: class_id in Supabase MUST match the YOLO model's class order. If your model says "Class 0: CHM_Uniform", then CHM must be class_id=0 in Supabase. They must be in the EXACT same order.
⬇️

✅ Step 8: Verify It Works

🔧 Troubleshooting

ProblemCauseFix
Low accuracy (mAP50 < 50%) Not enough photos or bad labels Add more photos, check boxes are drawn correctly
Wrong uniform detected Class IDs don't match Check class order in training vs Supabase class_id
Model won't load in browser ONNX export issue or CORS Re-export with simplify=True. Check CORS headers
"No YOLO model configured" warning YOLO_MODEL_URL is empty Update uniform.ts with the URL to your .onnx file
