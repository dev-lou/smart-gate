1. Abstract

The Smart Gate System is an intelligent access control solution designed for educational institutions. It replaces traditional manual gate security with an automated system that verifies students through face recognition and uniform compliance checking — all running entirely in the browser on a standard Android tablet.

Unlike traditional smart gate implementations that require a Raspberry Pi, Python backend, or cloud AI inference, this system achieves 90% software / 10% hardware by leveraging MediaPipe Tasks Vision for face detection, ArcFace ONNX for face recognition (99.4% LFW accuracy), and YOLO11n for custom uniform detection — all executed inside a Web Worker using ONNX Runtime Web with WebGPU acceleration.

The system follows a three-service monorepo architecture using Next.js 16, TypeScript, and pnpm workspaces: a kiosk PWA for the tablet gate, a guard station for student enrollment with multi-angle photo capture, and an admin dashboard for system configuration. Data syncs between IndexedDB (offline) and Supabase (cloud) when connectivity is available. CI/CD is handled through GitHub Actions with Turborepo-optimized builds, Dependabot for dependency updates, and CodeQL for security scanning.

2. Problem Statement

Traditional school gate security relies on manual visual inspection by security guards — an approach that is error-prone, inconsistent, and unscalable. Students can borrow IDs, impersonate others, or enter without proper uniform. Existing smart gate solutions require expensive hardware (Raspberry Pi, dedicated cameras, servers), complex Python-based AI pipelines, and constant internet connectivity.

The core problem this project addresses: How can we build a highly accurate, affordable, and offline-capable smart gate system using only a consumer tablet and minimal hardware?

❌

Current Pain Points

  • Manual ID checks are slow and unreliable
  • Uniform violations go unnoticed
  • No audit trail for entries
  • Guard fatigue leads to missed violations
  • Guest/visitor tracking is paper-based
  • Existing smart gates cost $500+
✅

Proposed Solution

  • Automated face recognition in 200-300ms
  • YOLO-based uniform compliance check
  • Complete digital audit trail
  • Runs on a $200 Android tablet
  • Works fully offline
  • 3-photo enrollment for high accuracy

6.3 Offline-First Design

How offline mode works

The kiosk uses IndexedDB as its primary data store, with Supabase as a sync target. All AI processing is local:

  1. On first sync: Download enrolled students from Supabase → store in IndexedDB → process photos → generate embeddings
  2. Face recognition: Looks up embeddings from IndexedDB (no network needed)
  3. Uniform detection: Downloads YOLO ONNX model once → cached by service worker
  4. Access logs: Written to IndexedDB immediately → synced to Supabase when online
  5. Settings: Downloaded from Supabase on sync → used from IndexedDB when offline
  6. Background sync: Triggers on init + network online events

6.4 Error Recovery

🛡️ ErrorBoundary

Wraps the entire kiosk in a React error boundary that catches runtime crashes and displays a friendly restart UI with the error message.

🔄 Arduino Auto-Reconnect

When the USB cable is bumped, the system automatically retries connection every 5 seconds for up to 60 seconds.

📸 Photo Processing Mutex

Uses a ref-based mutex to prevent concurrent photo processing when sync triggers while already processing.

📡 Graceful Degradation

If YOLO model isn't loaded → color-based fallback. If face recognizer fails → detection-only mode. If offline → local-only mode.

9. AI Pipeline — Detailed

9.1 Face Detection

Technology

MediaPipe Tasks Vision — BlazeFace Short Range model, a lightweight face detector optimized for mobile devices. Runs via WebAssembly (WASM) on the main thread.

Input: Full video frame (640×480)
Output: Bounding boxes as normalized ratios [x, y, w, h]
Speed: ~5ms per frame

Execution

// face.ts — Main thread
const vision = await FilesetResolver.forVisionTasks(
  "cdn.jsdelivr.net/.../wasm"
);
faceDetector = await FaceDetector.createFromModelPath(
  vision, FACE_DETECTOR_MODEL
);

// Usage in loop:
const result = faceDetector.detect(canvas);
return result.detections.map(det => ({
  bbox: [box.originX / w, box.originY / h,
         box.width / w, box.height / h],
  confidence: det.categories?.[0]?.score ?? 0,
}));

9.2 Face Recognition

Technology

ArcFace w600k_mbf — MobileFaceNet backbone with ArcFace loss, achieving 99.4% on Labeled Faces in the Wild (LFW) benchmark. Runs via ONNX Runtime Web in a Web Worker.

Input: 112×112 RGB face crop
Normalization: (pixel - 127.5) / 127.5 → [-1, 1]
Output: 512-dim L2-normalized embedding
Speed: ~30-50ms on WebGPU

Execution

// inference.worker.ts — Web Worker
const tensor = new ort.Tensor("float32",
  float32Data, [1, 3, 112, 112]);

const feeds = {};
feeds[session.inputNames[0]] = tensor;
const results = await session.run(feeds);

// L2 normalize
const embedding = new Float32Array(raw.length);
let sum = 0;
for (let i = 0; i < raw.length; i++) {
  embedding[i] = raw[i];
  sum += embedding[i] * embedding[i];
}
const norm = Math.sqrt(sum);
for (let i = 0; i < embedding.length; i++)
  embedding[i] /= norm;

9.3 Face Matching

The matching algorithm compares the query embedding against ALL enrolled embeddings per student (3 angles) using cosine similarity, taking the best score:

// face.ts — Cosine similarity matching
function matchFace(embedding, enrolledFaces) {
  let bestScore = -1, bestPerson = null;

  for (const enrolled of enrolledFaces) {
    for (const emb of enrolled.embeddings) {
      const score = cosineSimilarity(embedding, emb);
      if (score > bestScore) {
        bestScore = score;
        bestPerson = enrolled;
      }
    }
  }

  return {
    person: bestScore >= 0.6 ? bestPerson : null,
    confidence: bestScore,
    matched: bestScore >= 0.6,
  };
}

function cosineSimilarity(a, b) {
  let dot = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dot / (Math.sqrt(normA) * Math.sqrt(normB));
}

Uniform detection uses YOLO11n (custom-trained ONNX model) running in the Web Worker. If no model is loaded, a color-based fallback is used:

// 2. Open gate
